Google OAuth2 Device Code flow (“Connect YouTube”).

Mirrors the Twitch device flow in crate::oauth, but for a Google account so we can read the user’s YouTube subscriptions (youtube.readonly). Device code suits a desktop app: the user is shown a short code + URL, authorizes in a browser, and we poll for the token. Unlike Twitch, Google’s device flow needs both a Client ID and a Client Secret (a “TV and Limited Input devices” OAuth client created in Google Cloud Console); refresh keeps the same refresh token.

NOTE: live verification needs a Google Cloud OAuth client with the device flow enabled and the YouTube Data API turned on for the project.
