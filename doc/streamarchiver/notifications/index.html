Module notifications 

Source
Expand description

Desktop notifications for recording lifecycle events.

On Windows we build a rich toast directly via WinRT (Windows.UI.Notifications): the channel’s profile pic as a circular app-logo, its banner as the hero image, the channel name + stream title + game as text, and a “Watch stream” button that opens the channel URL (protocol activation — handled by Windows, no app callback needed). On other platforms we fall back to notify_rust.

Toasts are shown under the built-in Windows PowerShell AppUserModelID so they appear without registering our own AUMID / Start-Menu shortcut; that’s also why Windows attributes them to “Windows PowerShell”. Proper branding and buttons that call back into the app would need a registered AUMID + COM activator (a separate, packaging-adjacent piece).
