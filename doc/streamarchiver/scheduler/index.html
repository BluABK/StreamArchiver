Module scheduler 

Source
Expand description

Shared poll scheduler.

A single task loops over all enabled monitors, batches the due ones by detection method, runs detection (Twitch Helix in one batched call; scrape/generic probes concurrently with a cap), writes results back to the store, and emits an AppEvent::MonitorState on any state change. This is the low-idle-footprint design: one timer, batched work, no thread/process per channel.
