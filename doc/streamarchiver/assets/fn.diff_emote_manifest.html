Expand description

Diff two emote manifests by code (case-sensitive, as typed in chat) and return one AssetChange per added/removed code. Empty/whitespace codes are ignored (legacy name-less entries can never match chat anyway). An id-only change to an existing code yields nothing — only the code set matters here, so churn in ids/urls doesn’t spam the history. Output is sorted by code so the log (and the unit test) is deterministic.
