Structs§

AutoStart
Manages launch-on-login via the OS autostart mechanism (HKCU Run key on Windows), keyed to the current executable path.
DetachedJob
A named Win32 Job Object without KILL_ON_JOB_CLOSE. Closing our handle (app exit) does not terminate members, so an assigned download tool keeps running after we quit; the kernel keeps the named job alive while any member runs, so a later launch can open it by name to terminate the whole tree. Identity/liveness of the individual process is tracked separately via process_start_time + pid_alive.

The OS creation time of pid (FILETIME 100ns ticks), or None if it can’t be opened. Stored alongside the PID so a re-attach can reject a recycled PID (a different process now wearing the same number).
