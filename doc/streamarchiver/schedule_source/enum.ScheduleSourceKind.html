pub enum ScheduleSourceKind {
    TwitchSchedule,
    YouTubeScrape,

    TwitchBannerOcr,
    YouTubeCommunityOcr,
    TwitterPinned,
    Discord,
    OtherImageOcr,
}
Expand description

A method for resolving a channel’s upcoming schedule. The variant order here is irrelevant; user priority lives in the persisted SourceEntry list.

Variants§

§

TwitchSchedule

Twitch’s published schedule via Helix /schedule.

§

YouTubeScrape

YouTube upcoming livestreams scraped from the /streams page.

§

YouTubeApi

YouTube upcoming livestreams via the Data API (search.list + videos.list); self-gates on a configured API key.

§

TwitchBannerOcr

OCR the channel’s already-downloaded Twitch offline banner.

§

YouTubeCommunityOcr

OCR the image attached to the channel’s latest YouTube community post.

§

TwitterPinned

OCR the image on the channel’s pinned Twitter/X tweet.

§

Discord

Discord scheduled events (opt-in; uses the user’s Discord token).

§

OtherImageOcr

OCR a user-supplied image (per-channel path or URL).

Source

pub fn badge_icon(self) -> &'static str

Font-safe emoji badge shown on calendar items to indicate where the schedule came from. Glyphs are restricted to ones the app already renders in production (egui rasterizes only monochrome outlines from its bundled NotoEmoji subset + the OS fallback; see crate::models::ContentType::icon). 📡 broadcast schedule · 📺 YouTube · 📷 any image/OCR source · 💬 Discord.

Source

pub fn has_open_target(self) -> bool

Whether “Open source” can resolve a meaningful target for this kind (the origin page or image). Discord events have no per-event URL.

Source

pub fn risky(self) -> bool

Whether this source carries an account-risk / ToS warning.

Source

pub fn is_per_monitor(self) -> bool

Discord is resolved by a separate batch sweep, not the per-monitor walk.

Source

pub fn is_ocr(self) -> bool

Whether resolving this source spends OCR work — a download plus a multi-second, token-costing CLI call. These are gated to their own slow cadence in the refresh walk so a forced UI refresh (which re-runs the cheap API/scrape sources immediately) can’t re-OCR an image that was already processed minutes ago.

Source

pub fn applies_to(self, platform: Platform, cfg: &ChannelSourceConfig) -> bool

Whether this source can produce a schedule for a given monitor — its platform matches and any required per-channel config is present.
