Download a per-channel singular asset (icon / banner) into dir, preserving history — this is an archiver, so a profile pic / banner the channel later changes must not be lost.

dir/{stem}.{ext} always holds the latest version. When the freshly fetched image differs (byte-for-byte) from the current canonical file, the old one is moved into dir/history/{stem}_{retired_at}.{old_ext} before being replaced; retired_at is the unix time it was supplanted, so the history reads as a timeline. An identical re-download is a no-op (no spurious history entry).
