Expand description

Inputs to expand_template. Each field maps to a {…} variable; empty fields render empty. title/channel/video_id are resolved metadata (empty for live recordings or id-less methods); resolution/height/width/fps/ vcodec are actual media info (filled only when probing is enabled).

Fields§

§name: &'a str§title: &'a str§channel: &'a str§video_id: &'a str§quality: &'a str

Configured quality selector (e.g. 1080p60, best).

§resolution: &'a str§height: &'a str§width: &'a str§fps: &'a str§vcodec: &'a str§acodec: &'a str

Audio codec, e.g. “aac”, “opus” (empty when not probed or unknown).

§take: &'a str

Attempt number (per-monitor take count); empty for on-demand videos.

§games: &'a str

Distinct game/category names played during the recording, joined + length- capped. Only known after the stream ends, so it’s filled at the post-rename (empty for the initial capture name and for on-demand videos).

§tool: &'a str

Capture tool: “streamlink”, “yt-dlp”, “ffmpeg” (empty renders empty).

§mode: &'a str

Download mode: “live”, “sabr”, “dash”, “hybrid”, “hybrid-dash”, “direct”, “vod”, “chat”.

§platform: &'a str

Stream platform: “twitch”, “youtube”, “kick”, “generic” (empty renders empty).

§secs: i64

Capture-start time (unix secs) for {date}/{time}/{timestamp}.

§went_live: i64

When the broadcast went live (unix secs); 0 means unknown → {went_live_date}/{went_live_time} render empty.
