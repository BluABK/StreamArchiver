Build identity, baked in at compile time by build.rs.

build_id is stamped into every detached download’s registry row (detached_process.spawn_build) so a later launch knows exactly which build started an in-flight download and can apply per-build compat fixups when it re-attaches (see crate::compat). The local patch counter in APP_VERSION increments on every rebuild, so this distinguishes even successive dev builds of the same commit.

build_id
A stable identifier for the running build, e.g. 0.1.42/a1b2c3d4e-dirty. APP_VERSION is <major>.<minor>.<local-build-counter>; GIT_HASH is the short commit hash with a -dirty suffix when tracked files were modified.
