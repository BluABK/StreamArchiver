The internal event bus (core -> UI) and tray -> UI command channel.

The core emits AppEvents on a tokio::broadcast channel; the UI subscribes so it never has to hot-poll the core. The tray sends UiCommands on a plain mpsc channel and wakes the egui loop via Context::request_repaint.

Constants§

TOGGLEABLE_JOBS
Periodic jobs that can be enabled/disabled from the Background view, as (display name, settings key). The display name matches the mark_job name so the UI can join a toggle with its next-run estimate. Default: enabled (get_setting(key) != "0"); each loop skips its work when disabled.
